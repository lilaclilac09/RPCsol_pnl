The language of blockchain APIs

To get data from Helius, you don't visit a website — you send carefully formatted messages using a protocol called JSON-RPC.

Think of it like a postal service with a strict form template. Every letter (request) must follow the same format or it gets returned unread.

JSON-RPC REQUEST 
{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "getSignaturesForAddress",
  "params": [
    "7xK...mNp",
    { "limit": 1000 }
  ]
}

HTTP/2 — the highway upgrade

Most web requests use HTTP/1.1, which is like a single-lane road: one car at a time. This tool uses HTTP/2, which is like a multi-lane highway.

HTTP/1.1 — Serial

Request 1 ████████
Request 2 ████████
Request 3 ████████

Total time: 3× per-request time

HTTP/2 — Parallel

Request 1 ████████
Request 2 ████████
Request 3 ████████

Total time: 1× per-request time ⚡

The tool opens one TLS connection to Helius and sends up to 100 requests through it simultaneously. That single connection has a massive flow-control window to avoid bottlenecks.

The 100-request speed limit

More parallel requests = more speed, right? Not always. Flood Helius with 5000 simultaneous requests and they'll rate-limit you. The tool uses a semaphore to cap at exactly 100 concurrent requests.

rpc.rs 
// 100-concurrent-request semaphore
let sem = Arc::new(
    Semaphore::new(100)
);
// Each request acquires one permit
let _permit = sem.acquire().await;

Retry logic — never give up (gracefully)

Networks are unreliable. Helius sometimes returns errors. The messenger never crashes on a single failure — it uses exponential backoff to retry up to 5 times.

1

Try 1 — immediate

Send the request. If we get a 429 or 5xx error, don't panic — try again.

2

Try 2 — wait 100ms

Brief pause, then retry. The server might have just been momentarily overloaded.

3–5

Tries 3–5 — growing delays

Each retry waits twice as long: 200ms → 400ms → 800ms. By try 5, we've given the server real time to recover. If all 5 fail, then — and only then — return an error.

💡
Why this matters when you extend the tool

If you ask an AI to add a new API call to this codebase, make sure it uses the same retry pattern. A missing retry on a new function will cause flaky failures that are hard to debug — the kind of bug where "it works most of the time."
