Four Primitives

oliver.mjs is built from four tiny building blocks. Each does one thing perfectly. Together they make the pattern.

Meet the cast

Think of the Oliver pattern like a busy kitchen brigade. There are exactly four roles, each with one job. Nobody steps on anybody else.

🚦
makeSemaphore

The traffic light. Decides how many calls can be in-flight at once. If you have 12 slots and 20 requests, the first 12 go immediately; the rest queue politely.

🔁
withRetry

The persistent courier. If a request fails with a temporary error (network blip, rate limit), it waits a moment and tries again — up to 4 times, backing off a little longer each attempt.

⏱️
timed()

The stopwatch. Wraps any call and records how long it took, how many bytes came back, and whether it succeeded or failed. Pure transparency.

⚡
Promise.all

The conductor. Launches every task simultaneously and collects all the results once they finish. This is the engine room of parallelism.

How they coordinate — a group chat

When runAll() fires, here is what the four primitives say to each other:

P

S
makeSemaphore

Hold on — I have 12 slots available. All 9 requests can go. Releasing everyone. 🟢🟢🟢🟢🟢🟢🟢🟢🟢

R
withRetry

Wrapping each request. If Helius throws a 429 or connection drops, I will quietly back off and try again. You will never know it happened.

T

Stopwatch started on all 9. I will record how long each takes and how many bytes come back.

S
makeSemaphore

getBalance finished in 340ms — slot freed. getWalletHistory done at 610ms — another slot freed.

T

All 9 done. Logging results: 7 OK, 2 with notes. Wall time: 912ms. Sequential estimate: 5,400ms.

P

Results collected in input order. Returning array to the caller. Done. 🏁

How they nest inside each other

The four primitives wrap around each request like Russian dolls. Reading from outside in:

1
Promise.all fires everything simultaneously

All tasks start at the same moment. This is the outer shell — it does not care what is inside each task.

2
timed() wraps each task for instrumentation

Every task is wrapped in a stopwatch. If it succeeds, you get result + ms + bytes. If it fails, you get the error message instead of a crash.

3
withRetry wraps the actual network call

If the call gets a 429 (too many requests) or the connection drops, withRetry silently backs off and retries up to 4 times before giving up.

4
makeSemaphore gates entry to the network

Before any call touches the network, it must acquire a slot from the semaphore. This is the innermost guardian that prevents you from blasting 50 calls at once and getting banned.

🧱
Composability: the real superpower

Each primitive works independently. You can use Promise.all without a semaphore. You can use withRetry in a completely different context. The Oliver pattern is powerful because these small, focused pieces compose — you can mix and match them for any parallel-fetch scenario, not just Solana.
